- types -> kontrak yang bisa men-describe shape of data
- data kalau diproses dalam function -> kita butuh expected type dan input type data yang sama seperti expected yang diminta
- primitif data type:
	- string; number; boolean; null; undefined
- metode ngasih data type:
	- explicit -> secara eksplisit menyebut tipe datanya dari sebuah data
	- implicit -> depend on data inference (sesuatu yang diserap; artinya jadi langsung menyerap tipe data dari value-nya)
		- kalau pake `const` yang di-infer literal -> literally value datanya ("Randy")
		- kalau pake `let` di-inference sebagai string -> ada possibily untuk diubah
		  
		  ![Pastedimage20260618200538|334](https://raw.githubusercontent.com/Agus-AI-PE/notes-media/main/Pastedimage20260618200538.png)
		 ![Pasted image 20260618200624|336](https://raw.githubusercontent.com/Agus-AI-PE/notes-media/main/Pastedimage20260618200624.png)

- implicit paling sering dilakukan
- explicit dilakukan ketika sesuatu tidak jelas -> biasanya yang berhubungan dengan object -> entitas
- di ts ada type data non-primif -> yang bentuknya object
- interface -> salah satu metode membuat shape of data dari sebuah object
	- ```typescript
	  interface User {
			id: number;
			name: string;
			age: number;
			isActive: boolean
	  }
	  
	  interface Config {
			appName: string;
			appVersion: string;
			appUrl: string;
	  }
	  ```
- kalau gak dibikin interface, gak akan ada enforcement/autocomplete untuk membantu kita -> expected datanya harus sesuai kontrak
