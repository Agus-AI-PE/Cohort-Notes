#### ==*Common Pattern*==
- why pattern matter?
	- karena llm itu cuma bisa 1 fokus task 
	- tanpa patterns -> one giant promps akan sulit di-debug, sulit di-test, sering berubah behavior ketika tasknya grows

- dengan pattern
	- bisa di-split menjadi kecil-kecil

- di case prompt chaining punya pattern -> disebutnya caller -> function untuk call llm

##### ==Prompt Routing==
- beda task (req user) beda intruksi
- mengklasifikasikan req user, lalu mengirim req ke prompt yang tepat (semestinya) / INTENT
- routing -> mencegah 1 generic asisten meng-handle setiap task/kasus
- metode ini menambah latensi
	- karena ada pre-prosesnya (mengklasifikasikan dulu)
	- ngakalinnya pake model yang lebih rendah (cheap and fast) -> untuk extractedIntent
- ![Pastedimage20260623011015165|628](https://raw.githubusercontent.com/Agus-AI-PE/notes-media/main/Pastedimage20260623011015165.png)

> NOTE: perlu untuk di-log -> prompt yang dimasukin ke dalam proses extractedIntent
	>- dengan begitu kita akan tau kenapa output-nya halu
	>- ganti llm yang lebih bagus, frontier llm tidak serta merta menjadi better outputnya
	>- most of the time -> datang dari konteks req (apakah system prompt nya/intructions/ data yang dimasukin)

- ==context injection kalo di-inject negative (misal, overwrite system prompt dengan intruksi bahaya) dari user prompt bagaimana?==
	- dg konsep input-output guardrail di luar llm sebagai interseptor
	- ```
	  [ User Input ] ─> [ INPUT GUARDRAIL (Cek teks bahaya) ] ─> [ LLM (System Prompt) ] ─> [ OUTPUT GUARDRAIL (Cek kebocoran data) ] ─> [ User ] 
	  ```
		- input guardrail -> dijalankan sebelum req sampai pada main agent
			- ekstraksi dulu (extractedIntent) or klasifikasi apakah req mengandung prompt jailbreak ato tidak -> kalo iya, maka tolak
		- output -> dijalakan ketika llm sudah generate output dan siap di-serve ke user
			- cek untuk eksplisit kebocoran data -> bisa dg another llm call
	- guardrail -> deterministic (pasti); pake kalkulator untuk 1+1 hasilnya 2; regex; static string matching; if else  -> di dashboar kita bisa tentukan role A tidak bisa melihat data X-Z dan beberapa orang tertentu bisa melihat data A-Z
	- guardrail -> non-deterministic (probabilistik); pake chatGPT buat nulis puisi, tekan button "send promp" 2x hasilnya berubah-ubah -> SYSTEM_PROPMT
