#### ==*Common Pattern*==
- why pattern matter?
	- karena llm itu cuma bisa 1 fokus task 
	- tanpa patterns -> one giant promps akan sulit di-debug, sulit di-test, sering berubah behavior ketika tasknya grows

- dengan pattern
	- bisa di-split menjadi kecil-kecil

##### ==Prompt Chaining (CoT)==

- di case prompt chaining punya pattern -> disebutnya caller -> function untuk call llm

##### ==Prompt Routing==
- beda task (req user) beda intruksi
- mengklasifikasikan req user, lalu mengirim req ke prompt yang tepat (semestinya) / INTENT
- routing -> mencegah 1 generic asisten meng-handle setiap task/kasus
- metode ini menambah latensi
	- karena ada pre-prosesnya (mengklasifikasikan dulu)
	- ngakalinnya pake model yang lebih rendah (cheap and fast) -> untuk extractedIntent
- ![Pastedimage20260623011015165|628](https://raw.githubusercontent.com/Agus-AI-PE/notes-media/main/Pastedimage20260623011015165.png)

- !NOTE: perlu untuk di-log -> prompt yang dimasukin ke dalam proses extractedIntent
	- dengan begitu kita akan tau kenapa output-nya halu
	- most of the time -> datang dari konteks req (apakah system prompt nya/intructions/ data yang dimasukin)
