#### ==*ChatCompletions/Response*==
- completion -> single shot request ke model/llm -> program kita ngirim dan menerima generated output
- provider llm punya 2 jenis API
	- ChatCompletions -> API endpoint legacy tapi sudah jadi industry standar
	- Response -> API endpoint model yang baru cuma hanya ketika pake openAI
- fungsinya sama -> menerima input dan melakukan generate output (single shot)
- useful buat summarization; rewriting; classification; extraction
- llm call bisa fail
	- misal, halu -> jadi cost money karena return imperfect output seperti hallucination tetap di-charge
- llm call -> bukan bagian dari application logic. namun eksternal dependency
- llm endpoint semuanya sudah menyediakan ChatCompletions, tapi tidak semuanya menyediakan Response API shapes
- mana yang harus kita pake? 
	- jawabannya gunakan Chat Completions
	- ketika program kita ada integrasi dengan llm dan ketika ingin berpindah dari provider satu ke provider llm lain, Chat Completions tidak pernah berubah yang berubah cuma modelnya saja
	- kalau pake Response API yang berubah semua code implementasinya

- ini common di banyak project lama/baru -> industri standar

	  { role: "system", content: ""},

- anvia AI SDK -> AI SDK provider agnostic & typescript first
	- other AI SDK: Vercel AI SDK; Mastra; TanStack;
- daleman anvia -> menggunakan openAI raw SDK
- kenapa butuh anvia?
	- untuk menyeragamkan code
	- karena hampir semua SDK memiliki shape yang berbeda-beda
	- PR-nya kalau mau ganti provider, migrasinya harus nulis ulang semua code implementasinya dan akan banyak sekali problem di sana
	- anvia menawarkan solving problem itu dan product code tetap clean. cuma ganti di parameter model-nya aja -> shape-nya tetap sama


![Pastedimage20260622202750763|649](https://raw.githubusercontent.com/Agus-AI-PE/notes-media/main/Pastedimage20260622202750763.png)
-  ada 3 pattern dari @anvia/core
1. createCompletion() -> penggunaannya untuk freeform string/text 
	- dikonsumsi manusia
	- agentic/workflow/pipeline
2. createParsedCompletion() -> men-generate data yang terstruktur (array, object) -> perlu validasi runtime (zod)
	- dikonsumsi code/app/program lain
3. createCompletionStream()
	- digunakan untuk chatting
